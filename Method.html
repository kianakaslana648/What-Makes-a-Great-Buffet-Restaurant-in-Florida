Method

 

Overall, the analysis is achieved by data collection, data munging, and visualizations. Typically, data used for each visualization is from the same origin-the Yelp dataset, but a different subset by specific data processing based on the needs of visualizations.

2.  Data Munging and Design Choice

 

2.1.   Figure 1 and 2: Number of Restaurants by State and Category Bar plot

2.1.1.      Data Munging

To get the actual number of operating restaurants, we filter the yelp datasets based on “is_open” and “category” columns. And we do the same for all the charts in this project. Since there are about 7000 categories in the dataset, the “Number of Restaurants by Category” chart only shows the top 25 types.

 

2.1.2.  Decision Choices

We want to show the distribution of the number of restaurants against categorical variables. And a bar chart is a fundamental and simple tool to achieve that goal. Since the goals of both charts are the same to show an overview of “category” and “state”, we use the same design style to avoid adding complexity to simple information.

The goal of these two plots is easy, to show what kind of information we can get out of this data since we will be using “category” and “state” columns heavily in our following project. From the “Number of Restaurants by State” chart, we can see that Yelp does not give us data from all the states, but only 14 states and a majority of them are in Pennsylvania and Florida. From the “Number of Restaurants by Category” chart, we can see the distribution of restaurants is normal as what we would expect, and we can get information like country, restaurant type, food type, etc. from the “category” column.

 

2.2.  Figure 3: Choropleth plot for buffet restaurants

2.2.1.  Data Munging

From the Yelp dataset, we simply filtered restaurants with categories containing 'Buffet', did counting and computed the percentages for choropleth plots.

 

2.2.2.  Decision Choices

Since we are interested in buffet restaurants of the Yelp dataset, a choropleth plot of all the buffet restaurants is good to reveal the geospatial features of buffet restaurant distribution. We use the color scale for the percentages of each state region. It's easy to catch blank states for the Yelp dataset and percentages of the remaining states.

 

2.3.   Figure 4: Line and bar plot for Buffet Reviews by State

2.3.1.      Data Munging

After filtering the Florida buffet restaurants from the Yelp dataset, aggregate (group by state and month) data to compute the total monthly review counts and average stars for each state.

 

2.3.2.  Decision Choices

This plot is designed for showing the change in buffet restaurants’ review counts and stars with time in the United States. Colors are used to map the states, so we could also compare these changes happening in different states.

In the first line chart, it is easy to find Florida (pink line) with the most review counts after 2013, followed by Massachusetts (red line). Another obvious fact is that review counts in buffet restaurants in almost every state decreased significantly after 2019, which is probably caused by Covid-19.

Besides, the line chart is connected to the bar chart (the second one), which makes a bridge between review counts and rating stars of these restaurants in different states. Specifically, users could select the specific time in the line chart and see the comparison of the average stars of these restaurants in each state. Also, it is allowed to click specific bars (states) in the bar chart, and then observe the change in restaurants’ review counts in these states in another chart. It is interesting to compare the review counts and star change of buffet restaurants in states by the interactive method!

 

2.4.   Figure 5: Map for location of buffet restaurants

2.4.1.  Data Munging

For the Florida map, we filtered all the restaurants with categories containing 'Buffet' and located in the state of Florida. With the columns of restaurant names, latitudes, longitudes, stars, and addresses we were able to create the Florida map.

 

2.4.2.  Decision Choices

To have the first view of buffet restaurants in Florida, we believe the best choice is to show them on a map. Once we plot all the buffet restaurants as markers in the 'Open Street' style, the users would catch them just like when they are using any map app. When the users zoom in and hover over some marker, they would see the name and when they click on it, they would see the stars and address. This design is inspired by map apps.

 

2.5.   Figure 6: Restaurant Bubble Chart (Innovative View)

2.5.1.  Data Munging

This chart only uses buffet restaurants in Florida from the Yelp dataset. To add more dimensions to this view, we split Star into three categories “Star < 3”, “,3<=Star<4” and “Star >= 4”. 

 

2.5.2.  Decision Choices

As the last part of the buffet restaurant introduction section, we want users to get a chance to know about every single restaurant.

The goal of these two plots is easy, to show what kind of information we can get out of this data since we will be using “category” and “state” columns heavily in our following project. From the “Number of Restaurants by State” chart, we can see that Yelp does not give us data from all the states, but only 14 states and a majority of them are in Pennsylvania and Florida. From the “Number of Restaurants by Category” chart, we can see the distribution of restaurants is normal as what we would expect, and we can get information like country, restaurant type, food type, etc from the “category” column.

 

2.6.   Figure 7: Box plot for qualitative factors

2.6.1.  Data Munging

Qualitative factors are provided in the 'attribute' column from the original dataset. We simply extract those qualitative factors by dealing with the strings in the 'attribute' column. The factors participating in the boxplot are ‘Parking’(Existing parking place or not), ‘Reservation’(Can be reserved or not), ‘WIFI’ (Has WIFI or not), and ‘Price’ (Which section the price is in).

 

2.6.2.  Decision Choices

For the qualitative factors, especially binary factors, a reasonable choice would be the boxplots. Once we plot the distribution of stars within each value of factors, we would directly see the star difference of different values, which indicates the influence of the factor. Thus, we chose qualitative factors of parking, reservation, WIFI, price to create the boxplots.

 

2.7.  Figure 8: Parallel plot for quantitative factors

2.7.1.  Data Munging

Quantitative factors are word scores computed on the reviews using a library called 'spacy'. After averaging all the scores of the same word within reviews of the same restaurant, we have the quantitative factors for each restaurant. The score for a word represents how much customers focus on this topic.

 

2.7.2.  Decision Choices

For the quantitative factors, we believe the parallel coordinate plot is an innovative kind to attract users. Several axes are created representative of the quantitative factors and lines are used to connect the projection points of each dimension. From this plot, we can have an insight into the connection of different quantitative factors. Since we put the star axis in the first place, it's much easier to see the influence of quantitative factors on the stars.

 

2.8.  Figure 9: Heatmap for factor correlations

2.8.1.  Data Munging

Based on attributes in the original dataset (star, review counts) and extracted from reviews (food, environment, variety, drinks, service, and food), conduct a correlation analysis.

 

2.8.2.  Decision Choices

The correlation value in the heatmap is encoded with color. The positive correlation is represented by red, while blue means negative correlation, and darker color means higher correlation. Also, a tooltip is added to make users find the correlation value more conveniently.

From the plot, we can see that the food score and the service score seem to have a positive relationship. The pattern shows an upward trend. We can also see that the color becomes lighter on the right bottom part of the plot, which means that the food score contributes more to the overall satisfaction level of the restaurant rather than the service score. The hypothesis here is that since most the buffet restaurants are self-served, customers may not care too much about the service level.

 

2.9.  Figure 10: Upset plot for Buffet Restaurant Category Distribution

2.9.1.  Data Munging

From the original dataset, buffet restaurants in Florida are selected. After that, the category column is exploded due to it is nested in a list (a restaurant may include many category keywords). Then, remove categories that are not frequent (rank below 30 in terms of counts). Next, sort and list the categories of the same restaurant again, which allows the upset plot to show the distribution of buffet restaurants with different category combinations. 

 

2.9.2.  Decision Choices

The upset plot is aimed to provide us with information about the category distribution of Florida buffet restaurants and their overall ratings. The reason for using an upset plot is this type of plot allows the audience to capture the distribution of category combinations, while normal bar plots cannot achieve that. Specifically, each category is presented with a dot, so the line connecting dots means the category combinations of buffet restaurants. Next, the length of bars is reflected for the number of each category combination restaurant. To show evaluation in different types of buffets, color is used to map the level of rating. Based on the meaning of colors, green is represented good ratings, yellow represents the medium level, while red represents bad ratings.

Generally, the plot gives us an overview of the category distribution of Florida buffet restaurants and their ratings. As the plot shows, there are more than 20 kinds of buffet

restaurants in Florida with different category combinations. Traditional American, Chinese, and Japanese buffets are most popular in Florida, with mixed ratings, followed by Indian and Latin buffets with relatively better reviews. Also, both brunch buffet and bar buffet restaurants are common, but the former is more popular. As for food and drinks, seafood, sushi, pizza, and beer buffets are very hot! 

 

2.10.  Figure 11 and 12: Radar and bar plot for rating in country categories

2.10.1.  Data Munging

From the original dataset, buffet restaurants in Florida are selected. After that, the category column is exploded due to it is nested in a list (a restaurant may include many category keywords), and only keep categories with foods (seafood, sushi, pizza, steakhouses, barbeque, and vegetarian). Then, the dataset is joined with a score dataset that has records of rating in food, drink, service, environment, sauce, and variety for each restaurant. The data for the score dataset is extracted from text reviews using sentiment analysis. Next, group by category and aggregate the data to compute the count and average scores for the star, food, service, drink, environment, sauce, and variety. Finally, save the dataset in JSON format and CSV format to make the radar plot (rating for buffet restaurants in different food categories) and bar plot (buffet restaurants distribution in different food categories). 

 

2.10.2.  Decision Choices

D3 is used to create both the region radar plot and bar plot due to the great interactivity and flexibility. The reason for using a radar plot is that there are different evaluation aspects for buffet restaurants, and a radar plot could help the audience compare the buffet restaurants’ distinction in food, drink, environment, service, sauce, and variety quickly. Also, the color encoding is selected based on its match with the entity. Typically, Brazilian is encoded with brown, Halal is represented by green, Latin is encoded with orange, Indian is encoded with yellow, American is represented with blue, Japanese is represented with pink, and Chinese is represented by red. The bar plot is complementary to the radar plot, to provide information about how many restaurants the rating is based on for each category.

Comparing figure 11 and figure 12, an interesting finding is that there are more American, Chinese, and Japanese buffets than other types of buffest, but they perform worse than others in terms of reviews in food, service, environment, service, sauce, drink, and variety. Typically, people give a higher score for Brazilian, Halal, and Indian buffets, especially for their foods and environment. As for variety, Latin, American, and Chinese buffets receive more good reviews, while Halal buffets get significantly better comments on drinks.

2.12.  Figure 15: Box plot for food insights

2.12.1.  Data Munging

Several dummy variables are created to identify whether some foods (lamb, curry, beer, brownie, calamari, chili, crawfish, sushi) are mentioned by the restaurant reviews. Then, aggregation is conducted based on these dummy variables to get the different distribution of stars for restaurants.

 

2.12.2.  Decision Choices

The boxplots show the relationship between whether customers commented on specific food and the rating stars. The interactive methods provide the transition between the boxplot and the distribution of the values.

 

 

3.  Reflections

3.1.  From general restaurant analysis to Florida buffet restaurants

At first, the project is aimed to make a general analysis of “What makes restaurants good”. The purpose of it is to find some important factors for different kinds of restaurants. However, compared to a general restaurant analysis, we think diving deeper into a specific type of restaurant can provide more interesting insights and practical suggestions. In this case, buffet restaurants are a good analysis choice because they are a niche market with great popularity. In addition, it gives us more dimensions to analyze what makes a restaurant good because of the significantly distinctive food and services provided by buffet restaurants. Therefore, the analysis scope is narrowed down from restaurants in the United States to buffet restaurants in Florida.

 

3.2.  The limitation of the Yelp dataset

Yelp dataset provides us partial data for restaurants, which limits the scope of the analysis (82 buffet restaurants). To solve this problem, we try to use Yelp Fusion API to gather more data from Yelp, but it comes out of the new restaurant records we get lack text reviews, which cannot support the deep analysis. In this case, we decide to accept the limitation and conduct a small but detailed buffet restaurant analysis. Also, we double-check the Florida buffet restaurants using Google Maps and found a similarly small number of buffet restaurants.

 

3.3.  The alignment of visualization tool

So far, visualizations are created by different tools including d3, altair, plotly, ggupset, and ggplot. Multiple tools using allow us to create different plots conveniently, but it also makes some plots not very compatible when they are put on a website. If we could start the project from the scratch, we will reproduce each plot using d3, which will make plots fancier and align the format of plots.
